O que é HTML, CSS e JavaScript?

HTML

Linguagem de marcação usada para estruturar conteúdo na web.

<h1>Olá, mundo!</h1>

Use para estruturar textos, imagens, links e tabelas.

CSS

Linguagem de estilos que define a aparência do HTML.

h1 { color: blue; }

Use para cores, fontes, espaçamentos e layouts.

JavaScript

Linguagem de programação que dá interatividade às páginas.

alert("Olá, mundo!");

Use para criar animações, validações e aplicativos dinâmicos.

O que é HTML, CSS e JavaScript?

HTML

HTML (HyperText Markup Language) estrutura o conteúdo da página: títulos, parágrafos, imagens e links.

<h1>Olá, mundo!</h1>

Quando usar: sempre que precisar representar conteúdo, textos, listas, tabelas e links.

CSS

CSS (Cascading Style Sheets) define a aparência: cores, fontes, espaçamento e layout.

h1 { font-size: 2rem; }

Quando usar: para estilizar e criar responsividade visual da interface.

JavaScript

JavaScript adiciona interatividade: manipula comportamento e responde a ações do usuário.

document.querySelector('button').addEventListener('click', () => alert('Olá'));

Quando usar: para validações, requisições de dados, animações e lógica de aplicação.

Boas práticas profissionais

Acessibilidade (a11y)

Acessibilidade significa garantir que pessoas com diferentes capacidades consigam usar seu site: leitores de tela, navegação por teclado, contraste adequado e texto alternativo para imagens.

Exemplo prático

<img src="foto-equipe.jpg" alt="Equipe do projeto reunida em mesa redonda">
<button aria-label="Abrir menu de navegação">☰</button>
<a href="#main-content">Ir para o conteúdo principal</a>

Dicas rápidas

  • Inclua alt descritivo em todas as imagens informativas.
  • Use elementos nativos (botões, links) quando possível — já trazem acessibilidade por padrão.
  • Garanta foco visível e ordem lógica de tabulação (use tabindex apenas quando necessário).
  • Forneça textos alternativos e legendas para mídia.

Semântica

Marcação semântica melhora a legibilidade do código e a experiência de ferramentas (SEO, leitores de tela). Use <header>, <nav>, <main>, <section>, <article> e <footer> corretamente.

Exemplo prático

<main>
  <section>
    <article>
      <h2>Título do artigo</h2>
      <p>Conteúdo...</p>
    </article>
  </section>
</main>

Dicas rápidas

  • Use <article> para conteúdos independentes (ex.: posts, tutoriais).
  • Use <section> quando for uma divisão temática da página.
  • Evite usar <div> quando houver um elemento semântico adequado.

Responsividade

Responsividade garante que o site funciona bem em telas de diferentes tamanhos: mobile-first, imagens adaptativas e layouts fluidos.

Exemplo prático (imagem responsiva)

<picture>
  <source media="(min-width:800px)" srcset="banner-large.jpg">
  <source media="(min-width:400px)" srcset="banner-medium.jpg">
  <img src="banner-small.jpg" alt="Banner do curso de desenvolvimento web">
</picture>

Dicas rápidas

  • Projete pensando primeiro em mobile (mobile-first).
  • Use unidades relativas (%, rem, em) para tamanhos quando for estilizar com CSS.
  • Implemente imagens responsivas (<picture>, srcset).

Organização do projeto

Estrutura clara de pastas e padrões de nomes facilitam colaboração e manutenção. Mantenha assets organizados e módulos/componentes bem separados.

Exemplo de estrutura

meu-projeto/
├─ index.html
├─ README.md
├─ src/
│  ├─ components/
│  ├─ pages/
│  └─ main.js
├─ public/
│  ├─ images/
│  └─ fonts/
└─ .gitignore

Dicas rápidas

  • Documente o projeto (README com setup, comandos e padrões).
  • Use convenções de nomes consistentes (kebab-case, camelCase, etc.).
  • Separe responsabilidades: HTML (estrutura), CSS (estilo), JS (comportamento).

Versionamento (Git)

Versionamento guarda histórico do código, permite colaboração e reversão de mudanças. Git + plataforma remota (GitHub/GitLab/Bitbucket) é o padrão.

Exemplo prático — comandos básicos

# inicializar repositório
git init

# ver status
git status

# adicionar arquivos
git add .

# commit com mensagem
git commit -m "Mensagem curta e descritiva"

# enviar para remoto (branch principal 'main')
git push origin main

Fluxo recomendado (simples)

# criar branch para feature
git checkout -b feature/explicacao-acessibilidade

# trabalhar, commitar
git add .
git commit -m "Adiciona seção de acessibilidade"

# enviar branch e abrir Pull Request
git push origin feature/explicacao-acessibilidade

Dicas rápidas

  • Escreva mensagens claras: verbo no presente, resumo curto + descrição opcional.
  • Use branches para features/bugfixes e Pull Requests para revisão.
  • Inclua um .gitignore para não versionar arquivos gerados (build, node_modules).

Etapas do Processo:

  1. Ideia: Concepção, pesquisa de mercado, validação inicial
  2. Planejamento: Escopo, arquitetura, tecnologias, cronograma
  3. Design: UI/UX, protótipos, guia de estilo
  4. Desenvolvimento: Código, APIs, banco de dados
  5. Testes: Unitários, integração, usabilidade
  6. Otimização: Performance, SEO, acessibilidade
  7. Deploy: Publicação, monitoramento, backup
  8. Sucesso: Métricas, feedback, melhorias contínuas

Observação: O ciclo de melhorias (representado pelas setas de retorno) indica que o processo é iterativo e contínuo.

Panorama de Tecnologias Web

O desenvolvimento web envolve diversas áreas. Cada uma possui ferramentas e responsabilidades próprias. Entender o panorama ajuda a escolher melhor por onde começar ou qual caminho seguir.

Front-end

O front-end é a parte do sistema que o usuário vê e interage: interfaces gráficas, botões, formulários, menus.

Tecnologias comuns

  • HTML, CSS, JavaScript
  • Frameworks: React, Angular, Vue

Exemplo mínimo

<button onclick="alert('Olá!')">Clique aqui</button>

Back-end

O back-end é a lógica de negócio, que roda no servidor. Ele recebe requisições, processa dados e devolve respostas.

Tecnologias comuns

  • Node.js (JavaScript), Java (Spring), Python (Django/Flask), PHP (Laravel), C# (.NET)
  • APIs REST, GraphQL

Exemplo mínimo (Node.js)

const http = require('http');
http.createServer((req, res) => {
  res.end('Olá do servidor!');
}).listen(3000);

Bancos de Dados

Os bancos armazenam as informações do sistema. Podem ser relacionais (tabelas) ou não-relacionais (documentos, grafos).

Tecnologias comuns

  • Relacionais: MySQL, PostgreSQL, SQL Server
  • NoSQL: MongoDB, Redis, Cassandra

Exemplo mínimo (SQL)

CREATE TABLE usuarios (
  id INT PRIMARY KEY,
  nome VARCHAR(100)
);

DevOps

DevOps integra desenvolvimento e operações: automação de deploy, infraestrutura como código, integração contínua (CI) e entrega contínua (CD).

Tecnologias comuns

  • Docker, Kubernetes
  • CI/CD: GitHub Actions, GitLab CI, Jenkins
  • Infraestrutura: AWS, Azure, Google Cloud

Exemplo mínimo (Dockerfile)

FROM node:18
COPY . /app
WORKDIR /app
RUN npm install
CMD ["npm", "start"]

Testes

Testes garantem a qualidade e evitam erros. Podem ser unitários, de integração, funcionais ou automatizados.

Tecnologias comuns

  • Front-end: Jest, Cypress
  • Back-end: Mocha, JUnit, PyTest

Exemplo mínimo (JavaScript com Jest)

test('soma 2 + 2 = 4', () => {
  expect(2 + 2).toBe(4);
});

Como escolher seu caminho?

  • Se gosta de design e experiência do usuário → invista em Front-end.
  • Se prefere lógica e processamento de dados → siga para Back-end.
  • Se curte dados e performance → estude Bancos de Dados.
  • Se gosta de automação e infraestrutura → explore DevOps.
  • Se tem perfil detalhista e qualidade → foque em Testes.

Muitas vezes, desenvolvedores acabam sendo Full Stack (atuando no front e back-end), mas é importante ter uma base sólida antes de expandir.
